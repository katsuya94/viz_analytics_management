    README¶ ↑

    WARNING¶ ↑

    Understanding this app requires a reasonable understanding of Rails and Ruby.

    Setup¶ ↑

    Ruby Setup¶ ↑

    This project uses Ruby 2.0.0-p247 as defined in .rbenv-version, we recommend using rbenv to handle multiple versions of ruby. This project uses bundler to manage gems run gem install bundler to install. After installing these, install dependencies with bundle install. The versioning control system is git.

    Database Setup¶ ↑

    This project is designed to work with PostgreSQL.

    config/database.yml should look like

    development:
  adapter: postgresql
  encoding: unicode
  database: dbname
  pool: pool
  host: hostname
  port: port (usually 5432)
  username: dbusername
  password: dbpassword

# Warning: The database defined as "test" will be erased and
# re-generated from your development database when you run "rake".
# Do not set this db to the same as development or production.
test:
  adapter: sqlite3
  database: db/test.sqlite3
  pool: 5
  timeout: 5000

    After connecting the database run rake db:create Then run rake metrics:init

    Environment¶ ↑

    .env should look like

    RACK_ENV=development
PORT=3000
SECRET_KEY=keyGeneratedWithRakeSecret
GETTER_PASSWORD=getterServerPassword

    Make sure you generate a key with rake secret.

    Models¶ ↑

    Metric

    Uses Single Table Inheritance to exhibit different behavior for different classes. Encapsulates logic for querying/getting new Datum.

    Datum

    Holds a value and a timestamp corresponding to a Metric at a certain point in time.

    Recent

    Joins Metric to its most recent Datum by Company.

    Company

    Contains information for looking up company within files obtained from the Getter Server.

    User

    Used for login. Has an associated Company.

    Metrics¶ ↑

    The three categories of Metric are subclasses of Metric

    • Raw

    • Calculated

    • Percentile

    All metric definitions are made in app/models/definitions

    Create a new definition then run MetricName.singleton('description') from rails console to instantiate.

    Use rails g percentile MetricName to generate a percentile metric for MetricName.

    For examples of model definitions, look in the app/models/definitions directory

    Tasks¶ ↑

    • rake worker:update_raw then rake worker:update_calculated should be run every week. These may take a long time.

    • rake metrics:init Initializes the singleton database records for all definitions in app/models/definitions.

    Deployment¶ ↑

    Make sure the Heroku toolbelt is installed, as outlined in Heroku's documentation.

    Set config variables with heroku config:set VAR=value Setting the DATABASE_URL config variable in Heroku selects the database used for production. The other config variables should mirror .env

    After getting permissions and commiting changes, use git push heroku master to push changes live on viz-analytics.herokuapp.com

    Use heroku run [cmd] to run tasks on the server.
